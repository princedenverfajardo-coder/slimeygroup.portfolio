Game Flowchart

Phase 1: Initialization and Game States

Startup: The game begins by initializing Pygame and game settings.


Main Loop: The game enters a continuous loop that checks the current state: Loading, Game Over, or Playing.


Game Over Logic: If the state is "Game Over," it draws the game over screen and waits for the player to press "R" to reset the game to its initial state (3 lives, 0 score) and return to "Playing".



Phase 2: Core Gameplay Mechanics

When in the "Playing" state, the loop handles the following sequentially:


Events & Timers: Processes user input (move, fire, quit) and updates the projectile reload timer.


Dynamic Difficulty: Spawns objects based on score milestones. The "Danger" level increases with the score, ranging from 45% at 5,000+ to 75% at 20,000+.


Combat: Checks if the fire button was pressed and the weapon is reloaded. If so, it fires three projectiles with a spread and resets the reload timer.



Phase 3: Collision and Rendering


Projectile Hits: If a projectile hits a research point, the player earns double points, and both the point and projectile are removed.


Player Collisions: If the player collides with an obstacle, they lose a life. If lives reach zero, the state shifts to "Game Over".


Rendering: Finally, the loop draws all active elements (player, points, obstacles, and UI) to the screen and regulates the frame rate via Clock.tick(FPS).
